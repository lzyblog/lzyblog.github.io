jQuery实现QQ音乐播放器🥳功能实现

发表于2021-05-06|更新于2021-06-14|项目案例前端实例
|字数总计:3k|阅读时长:13分钟|阅读量:

歌曲播放

播放工具类封装

  • 为了防止index.js代码过多，我们可以封装成一个工具类
  • 新建一个player.js文件，在index.html中引入，注意要在index.js之前引入

播放暂停

  • 在 HTML 页面插入一个 audio 标签，用于播放音乐
  • 在index.js中引入 player 工具库，实例化一个 Player 对象，并且传入 audio 对象
  • 当按钮被点击时，调用播放音乐的方法
  • 实现播放 / 暂停音乐的方法，当创建音乐标签时，在标签中保存索引及音乐信息
  • 接收两个参数，分别为 index 和music 信息，定义一个遍历，用于记录当前索引，判断是否为同一首音乐

底部播放 / 暂停 / 上一首 / 下一首

  • 对象的 preIndex 与 nextIndex 方法用于处理索引

删除歌曲

  • 删除数据时需要注意的就是前台删除数据后，后台也无需保存数据。并且需要对标签保存的数据进行重新排序
  • 删除后需要注意删除的数据是否是正在播放的音乐的前边，如果是需要将索引 - 1。否则会出现下一曲跳歌的现象

切换歌曲信息

  • 更换歌曲信息涉及到的基本信息包括：右侧信息栏、图片、歌手、歌曲名、专辑名、底部控制条的名称、时间
  • 实现这个这个功能也很简单，只是一些元素的替换。调用位置有两处
    • 第一次获取歌曲时（成功）的回调
    • 初始化列表的第一个音乐
  • 当点击其他歌曲时

进度条的实现

  • 获取被点击位置距离窗口的位置
  • 获取默认距离窗口的位置
  • 被点击的位置减去默认距离窗口的位置
  • 点击进度条时，调整小圆点以及前景色的位置

进度条的拖动

  • 鼠标拖拽使用 mousemove 方法监听，但必须在鼠标按下后监听
  • 实现逻辑与点击一致
  • 鼠标抬起则释放 mousemove 事件即可

音乐时间、进度条同步

  • 通过 timeupdate 事件监听是否播放，正在播放时，会不断触发这个事件。在事件内部通过 duration 与 currentTime 获取当前时长和总时长。
  • 于是可以在 player 类中新增加方法
  • 在我们的 index.js 中只需要调用这个方法即可
  • 同样的为进度条新增方法 setProgress
  • 这样完成后，在主函数中计算出当前时间所占比例即可调用此方法进行设置
  • 因为使用了百分比进行修改元素，那么 css 的定位方式也需要修改一下

点击进度条与歌曲同步

  • 实现思路：
    • 计算出总时长除以已播放时长的比例
    • 将歌曲进度设置为歌曲时长乘以上一步的比例
  • 为了实现拖拽时声音能够继续播放，因此将设置的方法改到 mouseup 事件中

声音控制

  • 声音控制与进度条控制几乎一致
  • 接下来为 player 类新增方法即可

bug 修复

  • 音乐进度的跳转应为一个正常的数字
  • 音量同理且音量范围为 0~1
  • 拖拽超出范围

歌词设置

歌词解析并且加载

  • 解析创建一个新的类用于解析歌词
  • 接下来只需要在自己的 JavaScript 中初始化这个类并且调用方法即可
  • 创建好了之后，还需要对样式进行一点点小的修改

歌词同步

  • 歌词同步需要在监听播放的进度中去设置
  • 因为使用了 margin-top 滚动歌词，那么需要将 html 也该动
  • 为了实现切换歌曲时，歌词也可以切换，也需要在切换歌曲时将保存的歌曲信息也切换
