Redis🎯Redis五大数据类型

发表于2021-06-28|更新于2021-06-29|Java后端Redis
|字数总计:5k|阅读时长:24分钟|阅读量:

Redis-key

  • 在Redis中无论什么数据类型，在数据库中都是以key-value形式保存，通过进行对Redis-key的操作，来完成对数据库中数据的操
Redis命令 说明
exists key 判断键是否存在
del key 删除键值对
move key db 将键值对移动到指定数据库
expire key second 设置键值对的过期时间
ttl key 查看剩余时间
type key 查看value的数据类型

关于TTL命令

  • Redis的key，通过TTL命令返回key的过期时间，一般来说有3种：
    • 当前key没有设置过期时间，所以会返回-1.
    • 当前key有设置过期时间，而且key已经过期，所以会返回-2.
    • 当前key有设置过期时间，且key还没有过期，故会返回key的正常剩余时间.

关于重命名RENAME和RENAMENX

  • RENAME key newkey修改 key 的名称
  • RENAMENX key newkey仅当 newkey 不存在时，将 key 改名为 newkey 。

更多Redis命令参考：https://www.redis.net.cn/order/

五大数据类型

String字符串

  • String类似的使用场景：value除了是我们的字符串还可以是我们的数字！
    • 计数器
    • 统计多单位的数量
    • 粉丝数
    • 对象缓存存储！
命令 说明
append key str 追加字符串，如果当前key不存在，就相当于setkey
strlen key 获取字符串的长度
命令 说明
incr key 键值自增1
decr key 键值自减1
incrby key 10 可以设置步长，指定增量为10
decr key 5 可以设置步长，指定减增量为5
命令 说明 实例
getrange key start end 获取指定范围的键值 getrange key1 0 3
setrange key offset value 从offset开始替换为指定的value SETRANGE key2 1 xx
命令 说明 实例
setex (set with expire) 设置过期时间 setex key3 30 “hello”
setnx (set if not exist) 不存在在设置,在分布式锁中会常常使用！ setnx mykey “MongoDB”
命令 说明
mset k1 v1 k2 v2 k3 v3 同时设置多个值
mget k1 k2 k3 同时获取多个值
msetnx k1 v1 k4 v4 msetnx 是一个原子性的操作，要么一起成功，要么一起失败
命令 说明
getset 组合命令，如果不存在值则设置，存在会覆盖原值

List列表

  • 在Redis里面，我们可以把list玩成 栈、队列、阻塞队列！ 所有的list命令都是用l开头的，Redis不区分大小命令
    • list实际上是一个链表，before Node after ， left，right 都可以插入值
    • 如果key 不存在，创建新的链表
    • 如果key存在，新增内容
    • 如果移除了所有值，空链表，也代表不存在！
    • 在两边插入或者改动值，效率最高！ 中间元素，相对来说效率会低一点~
命令 说明
lpush list value 将一个值或者多个值，插入到列表头部 （左）
rpush list value 将一个值或者多个值，插入到列表尾部 （右）
命令 说明
lpop value 移除list的第一个元素
rpop value 移除list的最后一个元素
命令 说明
lindex list index 获取指定index的元素
llen list 返回列表的长度
命令 说明
lrem list numer value 移除指定个数number的value值
命令 说明
trim mylist start end 通过下标截取指定的长度，只保留下标1和2的元素
命令 说明
rpoplpush oldlist newlist 移除列表的最后一个元素，将他移动到新的列表中！
命令 说明
lset list index newstr 将列表中指定下标的值替换为另外一个值，更新操作
命令 说明
linsert list before|after value 将某个具体的value插入到列把你中某个元素的前面或者后面！

set集合

  • set集合中的值无序且不能重复，命令以s开头
  • string和list的元素都是value，set中是member
命令 说明
sadd myset value set集合myset中添加元素
smembers set 查看集合所有值
sismember set value 判断某一个值是不是在集合中
命令 说明
scard myset 取集合中元素个数
srem myset value 移除集合中的元素

输入命令前几位按Tab会自动补齐

命令 说明
srandmember myset 从集合中随机抽取元素
srandmember myset number 指定随机抽取元素的个数
命令 说明
spop myset 随机移除集合中的元素
命令 说明
smove myset1 myset2 value 将myset1中的value移到到myset2中
命令 说明
sdiff key1 key2 差集
sinter key1 key2 交集
sunion key1 key2 并集

Hash哈希

  • Redis hash 是一个键值对集合，其命令以h开头
  • Redis hash是一个string类型的field和value的映射表，hash特别适合用于存储对象
命令 说明
hset myhash field value set一个key-value
hget myhash field 获取指定field的值
hmset myhash field1 value1 field2 value2 set多个key-value
hmget myhash field1 field2 获取多个值
命令 说明
hdel myhash field1 删除指定的字段filed1，其对应的value1也删掉了
hlen myhash 查询hash的字段数量
hexists myhash field1 判断指定字段是否存在
hkeys myhash 只获取所有的field
hvals myhash 只获取所有value
命令 说明
hincrby myhash field 3 指定对数值类型的增量
hsetnx myhash field value 如果不存在则可以设置，存在则不能设置

Zset有序集合

  • Redis zset 和 set 一样也是string类型元素的集合，且不允许重复的成员。命令以z开头
  • 不同的是每个元素都会关联一个double类型的分数。redis正是通过分数来为集合中的成员进行从小到大的排序。zset k1 score v1
  • zset的成员是唯一的，但分数score却可以重复
命令 说明
zrangebyscore salary min max 指定排序访问main~max
zrangebyscore salary -inf +inf 排序从负无穷大到正无穷大
zrangebyscore salary -inf +inf withscores 排序从负无穷大到正无穷大并带上scores值
ZREVRANGE salary 0 -1 改变顺序，反转
命令 说明
zrem myset value 移除指定集合的元素
命令 说明
zcount myset 1 3 获取指定区间的成员数量
